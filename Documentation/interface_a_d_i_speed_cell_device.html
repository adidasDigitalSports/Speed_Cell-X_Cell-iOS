Detailed Description

The ADISpeedCellDevice class represents a remote SpeedCell device.

Use this class to connect to and interact with the device. You can initialize an ADISpeedCellDevice from an ADIBluetoothDeviceSpeedCell object, that holds the necessary CBPeripheral object for the bluetooth communication.

You have the opportunity to subclass ADISpeedCellBindingLayer class and initialize an ADISpeedCellDevice object with it. In this case you can override the inner behaviour of ADISpeedCellBindingLayer.

// Download Sessions Example
// This is an example for downloading sessions from the SpeedCell device.
// With this function files can only be downloaded once.
@implementation DownloadSessionsExampleViewController {
ADISpeedCellDevice *_speedCellDevice;
}
// Download sessions with progress
- (void)downloadAllSessions {
// You should disable user interaction during download
// Use weakSelf inside the block
__weak typeof(self) weakSelf = self;
[_speedCellDevice setHandlerForDownloadProgressChange:^(NSInteger progress) {
// Progress is between 0 and 100
// Update your UI
}];
// Enable user interaction// Save or display downloaded sessions[_speedCellDevice downloadSessionsBinary:^(NSArray *rawSessions, NSError *error) {
if (error == nil) {
}
}];
}
@end
// Manage Device Time Example
// This is an example for getting and setting time on the SpeedCell device.
// It is assumed that the device is set up correctly and is connected.
@implementation ManageDeviceTimeExampleViewController {
ADISpeedCellDevice *_speedCellDevice;
}
- (void)getDeviceTime {
// Use weakSelf inside the block
__weak typeof(self) weakSelf = self;
[_speedCellDevice getDeviceTime:^(NSDate *dateTime) {
// Display or log acquired time
}];
}
// Setting the time to current date and time
// Important Notice: due to the nature of SpeedCell communication setTime operation stops the current workout, then waits for synchroniztion, so it takes quite some time.
// You should disable your viewcontroller's buttons and wait for the following callback before re-enabling interaction.
- (void)setDeviceTime {
// Use weakSelf inside the block
__weak typeof(self) weakSelf = self;
[_speedCellDevice setDeviceTime:[NSDate date] success:^(BOOL success) {
// Re-enable user interaction
}];
}
@end
// Real Time Stride Rate Example
// This is an example for getting and displaying stride rate coming from the SpeedCell device.
// It is assumed that the device is set up correctly and is connected.
//
// SpeedCell sends speed and stride rate readings in every second, if the device is in action.
@implementation HeartRateExampleViewController {
ADISpeedCellDevice *_speedCellDevice;
}
// Response is handled in a completion block, that you must provide
- (void)setupStrideRate {
// Use weakSelf inside the block
__weak typeof(self) weakSelf = self;
[_speedCellDevice startStrideRateUpdatesWithHandler:^(NSObject<ADIRSCMeasurementProtocol> *strideRateData, NSError *error) {
// Display or log the incoming heart rate
}];
}
@end

- (void) downloadSessions: (void(^)(NSArray *sessions, NSError *error))  completion

Downloads every session from the remote device.

Parameters
completionBlock object to be executed on completion of download. This block has no return value and takes two arguments: an NSArray of ADISpeedCellSession instances and an NSError containing the error. On success the error is nil.

Reimplemented from <ADIDownloadSessionsProtocol>.
